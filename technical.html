Bone Trail Technical Case Study

마케팅 페이지와 분리한 기술 소개입니다. 현재 구현은 Java/libGDX 기반의 턴제 던전 프로토타입이며, 핵심 규칙을 렌더링과 분리해 테스트하고 AI-assisted 작업을 증거 기록으로 묶습니다.

기술 소개의 역할

이 페이지는 게임을 파는 문장이 아니라, 구현 방식과 검증 방식을 설명하는 포트폴리오 표면입니다. 공개 플레이 빌드, Android release, store readiness, user acceptance는 아직 주장하지 않습니다.

강점은 완성된 화면보다 개발 방식에 있습니다. deterministic model test, UI runtime spine proof, evidence ledger, handoff record가 각각 구현과 협업의 과장을 막는 장치로 쓰입니다.

현재 공개 HTML은 `Bone Trail`이라는 포트폴리오 이름을 쓰지만, 내부 소스 repo/package 이름은 별도 거버넌스 변경 전까지 유지됩니다.
Engine Java / libGDX Android-first target, desktop review path.
Runtime split Model before render Gameplay rules remain testable without drawing.
Validation Headless proof Focused Gradle tests and render smoke checks.
Status In progress Prototype evidence, not shipped-game evidence.

구현 구조

현재 포트폴리오에 안전하게 말할 수 있는 기술 축만 남겼습니다.

01

Playable rule spine

이동, 전투, 아이템, class active, warp 흐름을 작은 던전 단위에서 검증하는 review capture입니다.

02

Deterministic tests

룰 검증은 화면 렌더링보다 먼저 실행됩니다. seeded RNG와 headless model tests를 기본 전제로 둡니다.

03

UI runtime spine

창, 버튼, modal dispatch, measured text 같은 UI 동작을 behavior harness로 먼저 고정하는 단계입니다.

검증 근거

공개 문구는 repo-local evidence ledger와 validation record를 기준으로 제한합니다.

Area Evidence Public claim boundary
Game rules Model tests, level validation, combat simulation harness records. Prototype rule validation. Not a shipped or balanced game claim.
UI modernization Source-backed UI/Text absorption docs and R2 UI runtime spine harness. First runtime-spine proof. Not polished UI or user acceptance.

AI 협업 구조

빠른 생성보다 중요한 기준은 과장 방지와 다음 세션 복구 가능성입니다.

Evidence-first workflow

작업 단위는 지시서, 진행 기록, handoff, status JSON, validation result로 남깁니다. 기능이 추가되면 먼저 어떤 증거로 주장할 수 있는지 구분하고, user acceptance가 필요한 항목은 별도로 둡니다.

이 방식은 포트폴리오에서도 그대로 쓰입니다. 공개 페이지는 게임 상세, 기술 페이지는 구현/검증 사례로 나누고, 두 페이지 모두 현재 증거로 말할 수 없는 항목을 주장하지 않습니다.

What remains

다음 강한 외부 증거는 30-60초 desktop run 영상과 더 직접적인 UI interaction proof입니다. 특히 interactive bag, quickslot, viewport behavior가 개선되어야 기술 설명이 실제 사용감으로 이어집니다.

기술 소개는 지금까지의 구현과 방향성을 보여주는 별도 표면입니다. 최종 서사와 가이드 문구는 별도 content gate에서 확정합니다.

아직 주장하지 않는 것

현재 페이지의 신뢰도를 지키기 위해 명시적으로 제외한 주장입니다.

  • Shipped game공개 플레이 빌드나 스토어 출시가 완료된 상태가 아닙니다.
  • Polished UI현재 UI는 user accepted quality로 잠그지 않았습니다.
  • Closed ownershipGPLv3 reference-source route와 관련된 권리 경계를 과장하지 않습니다.
